# プロダクト要件定義

## 1. 概要

| 項目 | 内容 |
|---|---|
| アプリ名 | TripPlan |
| 種別 | 旅程管理・共有 Web アプリケーション |
| 対象ユーザー | 個人旅行者 |
| デザインテイスト | モダン・スタイリッシュ（ダーク/ブックモード） |
| 目的 | 旅行の日程・訪問スポット・費用を一元管理し、URLで仲間に共有できるようにする |

---

## 2. データモデル

詳細な型定義は [data-model.md](../architecture/data-model.md) を参照。

### 2.1 Trip（旅行）

| フィールド | 型 | 説明 |
|---|---|---|
| id | number | 旅行の一意なID |
| title | string | 旅行タイトル（例: 京都・大阪の旅） |
| destination | string | 目的地 |
| startDate | string (YYYY-MM-DD) | 出発日 |
| endDate | string (YYYY-MM-DD) | 帰宅日 |
| cover | string (emoji) | サムネイル用絵文字アイコン |
| days | Day[] | 日程の配列（出発日〜帰宅日の日数分を自動生成） |

### 2.2 Day（日程）

| フィールド | 型 | 説明 |
|---|---|---|
| date | string (YYYY-MM-DD) | 日付 |
| spots | Spot[] | その日に登録されたスポットの配列（順序あり） |

### 2.3 Spot（スポット）

| フィールド | 型 | 説明 |
|---|---|---|
| id | number | スポットの一意なID |
| name | string | スポット名（必須） |
| startTime | string (HH:MM) | 開始時刻。先頭スポットのみ直接保持し、以降は自動算出 |
| duration | number (分) | 所要時間。30/60/90/120/150/180/240/300分から選択 |
| category | string | カテゴリ（観光／グルメ／散策／体験／宿泊） |
| note | string | メモ（任意） |
| url | string | 参考URL（任意） |
| costs | Cost[] | 費用項目の配列（**最大3件**） |

### 2.4 Cost（費用項目）

| フィールド | 型 | 説明 |
|---|---|---|
| label | string | 費用の項目名（例: 入場料） |
| amount | string (数値文字列) | 金額（円） |

- 1スポットにつき費用項目は **0〜3件** まで登録可能
- 各スポットの合計費用は `costs` 配列の `amount` の合計値として算出

## 3. 画面構成

### 3.1 ホーム画面（旅行一覧）
- 登録済みの旅行をカード形式で一覧表示
- 各カードに: アイコン・タイトル・目的地・期間・日数を表示
- 「+ 旅行を追加」ボタンで旅行作成画面へ遷移
- 旅行が0件の場合は空状態メッセージを表示

### 3.2 旅行作成画面
- 絵文字アイコン選択（10種類から選択）
- 旅行タイトル入力
- 目的地入力
- 出発日・帰宅日入力（日付ピッカー）
- 「旅行を作成」ボタン押下で `days` を自動生成し、詳細画面へ遷移

### 3.3 旅行詳細画面

**3.3.1 ヘッダー**
- 旅行アイコン・タイトル表示
- 「共有」ボタン（共有モーダルを開く）
- 目的地・総日数の表示

**3.3.2 日付タブ**
- 「▶N日目・曜日（月/日）」形式のタブで日を切り替え

**3.3.3 表示タブ**
- 「スケジュール」「地図」の2タブで表示切替

#### スケジュールタブ
- その日のスポットを開始時刻順にリスト表示
- 各スポット行に表示する情報:
  - 開始時刻〜終了時刻
  - スポット名
  - カテゴリバッジ
  - 所要時間バッジ
  - メモ（あれば）
  - 合計費用（費用が設定されていれば金額を表示）
  - 🔗 リンクアイコン（URLが設定されていれば）
- 各行はドラッグ&ドロップで並び替え可能（詳細は §4）
- 行をクリックするとスポット詳細モーダルを表示
- 行右端の「×」ボタンでスポットを削除
- リスト下部の「+ スポットを追加」ボタンでインライン追加フォームを展開

#### 地図タブ
- 目的地名とその日のスポットを簡易リスト形式で表示
- 各スポットの名称・時間帯・所要時間・カテゴリを表示

### 3.4 スポット追加フォーム（インライン）
画面遷移せずスケジュールタブ内に直接展開する。入力項目:

| 項目 | 入力形式 | 必須 |
|---|---|---|
| スポット名 | テキスト入力 | ✓ |
| 開始時刻 | タイムピッカー（先頭スポットの場合のみ意味を持つ） | ✓ |
| 所要時間 | セレクトボックス（30分〜5時間） | ✓ |
| カテゴリ | セレクトボックス | ✓ |
| URL | テキスト入力 | — |
| 費用（最大3件） | 項目名＋金額の組み＋「追加」で増減 | — |
| メモ | テキスト入力 | — |

- 「追加する」ボタンでスポットを登録し、フォームを閉じる
- 「キャンセル」ボタンで入力内容を破棄してフォームを閉じる

### 3.5 スポット詳細モーダル
スポット行クリックで表示するモーダル。表示内容:
- スポット名・カテゴリバッジ
- 開始時刻〜終了時刻・所要時間
- URL（設定時のみ、クリックで新規タブで開く）
- 費用一覧（項目ごとの金額・合計金額、未設定時は非表示）
- メモ（未入力時は「なし」と表示）
- 「閉じる」ボタン、または背景クリックで閉じる

### 3.6 共有モーダル
- 共有用URL（`https://tripplan.app/share/{tripId}`）を表示
- 「リンクをコピー」ボタンでクリップボードにコピー
- 「閉じる」ボタンまたは背景クリックで閉じる

## 4. ドラッグ&ドロップと時刻再計算

詳細は [time-recalc.md](../architecture/time-recalc.md) を参照。

### 4.1 操作方法
- スポット行をドラッグして任意の位置にドロップすると順番を入れ替え
- ドラッグ中の行は半透明表示、ドロップ候補の行は下線ハイライト

### 4.2 時刻再計算ルール（`recalcTimes`）
並び替え後、以下のルールで全スポットの開始時刻を再計算:

1. **先頭スポットの開始時刻**を基準点とする
2. 2番目以降の各スポットの開始時刻 = **直前スポットの開始時刻 + 直前スポットの所要時間**
3. この計算をリストの末尾まで連鎖的に適用する

### 4.3 発火タイミング
- ドラッグ&ドロップによる並び替え後
- スポットの追加後
- スポットの削除後

## 5. カテゴリ定義

| カテゴリ | 用途例 | バッジカラー |
|---|---|---|
| 観光 | 寺社・公園など | インディゴ系 |
| グルメ | 飲食店・市場など | アンバー系 |
| 散策 | 街歩き・自然散策など | グリーン系 |
| 体験 | アクティビティ・乗り物体験など | ピンク系 |
| 宿泊 | ホテル・旅館など | グレー系 |

---

## 6. 非機能要件・制約

- データはブラウザリロードでリセットされることを想定（クライアントサイドのみで保持）
- 費用入力は数値のみ許容。未入力の場合は金額非表示（「—」表示）とする
- URLは未入力でも他項目の登録に影響しない
- 1スポットあたりの費用項目は3件を超えて追加できない（UI上で「+ 追加」ボタンを非表示にする）
